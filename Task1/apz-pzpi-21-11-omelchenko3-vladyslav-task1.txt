1.	Бізнес-вимоги
1.1.	Передумови

Зі збільшенням популярності здорового способу життя та спорту, зростає і кількість спортивних залів. Власники залів стикаються з проблемами управління ресурсами, моніторингу умов у приміщеннях та забезпеченням якісних послуг для відвідувачів. Сучасні відвідувачі очікують високого рівня обслуговування, безпеки та комфорту під час тренувань. Для цього необхідно забезпечити ефективне управління залами, своєчасне обслуговування обладнання та моніторинг умов у залах, що потребує розробки відповідної програмної системи.
Розробка програмної системи для автоматизації управління спортивними залами та моніторингу умов виходить з необхідності підвищення ефективності управління ресурсами та забезпечення комфорту і безпеки для відвідувачів. Система дозволить власникам залів краще керувати своїм бізнесом, а тренерам – зосередитись на наданні якісних послуг без зайвого адміністрування. Відвідувачі ж отримають можливість легко реєструватись на тренування, відміняти їх, а також мати впевненість у комфортних умовах під час занять.
Цілі та завдання програмної системи для автоматизації управління спортивними залами та моніторингу умов полягають у створенні інструменту, що забезпечить ефективне управління залами, полегшить адміністрування для власників та тренерів, а також забезпечить комфорт і безпеку для відвідувачів.

1.2.	Бізнес-можливості

Програмна система для автоматизації управління спортивними залами та моніторингу умов вирішує ключові проблеми, з якими стикаються власники та менеджери спортивних закладів. В даний час ринок автоматизації управління спортивними об'єктами перебуває на стадії зростання, оскільки все більше людей займаються спортом і відвідують тренажерні зали.
В ході моніторингу існуючих систем можна виділити дві наступні, які будуть значними конкурентами майбутнього продукту:
Першим з них є GymMaster. Це програмне забезпечення для управління спортивними залами, яке автоматизує процеси реєстрації, оплати та управління розкладом тренувань. Система також дозволяє відстежувати присутність відвідувачів і інтегрується з системами контролю доступу.
Серед переваг даної системи можна виділити наступні:
•	Автоматизація реєстрацій та оплати, що спрощує адміністрування.
•	Управління розкладом тренувань та відстеження присутності.
•	Інтеграція з системами контролю доступу для забезпечення безпеки.
Також у цієї системи було виділено наступні недоліки:
•	Відсутність функціоналу для моніторингу умов у спортивних залах (рівень шуму, температура, вологість повітря).
•	Обмежені можливості для створення індивідуальних звітів та аналізу даних.
•	Відсутність розширених функцій для тренерів, таких як можливість додавання, редагування та видалення тренувань безпосередньо через систему.
У порівнянні з GymMaster, майбутня система пропонує розширений функціонал для моніторингу умов у спортивних залах, що дозволяє забезпечити оптимальні умови для тренувань. Також вона надає більш гнучкі можливості для тренерів та розширений функціонал для створення і редагування тренувань.
Іншим конкурентом буде система Mindbody. Це комплексне програмне забезпечення для управління фітнес-центрами, студіями йоги та іншими спортивними закладами. Система включає функції для управління розкладом, реєстрації клієнтів, обробки платежів та маркетингових кампаній.
До переваг Mindbody можна віднести:
•	Комплексний підхід до управління спортивними закладами, включаючи маркетингові інструменти.
•	Інтеграція з різними платіжними системами для зручної обробки платежів.
•	Потужні інструменти для управління розкладом та реєстрацією клієнтів.
З основних недоліків системи виділено наступні:
•	Відсутність функціоналу для моніторингу умов у спортивних залах.
•	Обмежена можливість кастомізації інтерфейсу та функціоналу під специфічні потреби закладу.
•	Висока вартість використання для малих та середніх спортивних закладів.
На відміну від Mindbody, майбутня система буде включати функціонал для моніторингу умов у залах, що забезпечує комфорт та безпеку для відвідувачів. Крім того, наша система надасть тренерам можливість самостійно додавати та редагувати тренування, що зробить її більш гнучкою та зручною для використання.
На ринку існують потужні системи управління спортивними закладами, такі як GymMaster та Mindbody, які пропонують широкий спектр функцій для автоматизації процесів управління. Однак, жодна з них не пропонує комплексний моніторинг умов у залах, що є важливим для забезпечення комфорту та безпеки відвідувачів. Наша система не лише закриває цю прогалину, але й надає розширені можливості для тренерів та власників залів, роблячи управління більш ефективним та зручним.
 
2.	Концепція рішення
2.1.	Окреслення концепції

Метою проекту є розробка програмної системи для автоматизації управління спортивними залами та моніторингу умов, спрямованої на підвищення ефективності управління, покращення комфорту для відвідувачів та забезпечення безпеки тренувань. Цільова аудиторія включає власників спортивних залів, тренерів та відвідувачів. Програмний продукт розробляється з метою надання користувачам зручного інструменту для автоматизації процесів, моніторингу умов у залах та покращення загального рівня сервісу.
Впровадження програмної системи для управління спортивними залами та моніторингу умов дозволить користувачам легко та ефективно керувати своїми закладами, переходячи від ручного управління до автоматизованої системи. Процеси реєстрації, оплати, управління розкладом та моніторинг умов стануть більш доступними та точними. Це сприятиме не лише збереженню часу та зусиль персоналу, але й покращить загальний рівень обслуговування клієнтів, сприяючи створенню комфортного та безпечного середовища для тренувань.
Розроблений функціонал програмної системи дозволить ефективно вирішувати повсякденні завдання, пов’язані з управлінням спортивними залами. Система включатиме інструменти для моніторингу умов у залах, такі як температура, вологість і рівень шуму, що забезпечить оптимальні умови для тренувань. Власники залів зможуть автоматизувати процеси реєстрації та оплати, зменшуючи адміністративні витрати. Тренери отримають можливість легко створювати та редагувати тренування, а відвідувачі – зручно реєструватися на заняття та відміняти свої записи.

2.2.	Головна функціональність

Основні функції системи для автоматизації управління спортивними залами та моніторингу умов:
MF-1: Реєстрація користувача у системі;
MF-2: Створення нового спортивного залу та налаштування його зон;
MF-3: Редагування існуючих спортивних залів;
MF-4: Видалення існуючих спортивних залів;
MF-5: Перегляд інформації про наявні спортивні зали та їх зони;
MF-6: Додавання тренерів до спортивного залу;
MF-7: Формування розкладів для спортивного залу;
MF-8: Створення нових тренувань тренерами;
MF-9: Редагування існуючих тренувань тренерами;
MF-10: Видалення існуючих тренувань тренерами;
MF-11: Бронювання користувачами місць на певні тренування;
MF-12: Додавання датчиків шуму, температури та вологості до зон спортивного залу;
MF-13: Перегляд поточних показників шуму, температури та вологості у зонах залу;
MF-14: Доступ до особистого кабінету користувача та редагування особистої інформації;
MF-15: Сортування та фільтрація всіх списків даних, що надаються користувачу.

3.	Рамки та обмеження проєкту
3.1.	Рамки первинного випуску

Функції серверної частини:
•	Автентифікація та авторизація: Сервер має забезпечити систему автентифікації користувачів та надання прав доступу на основі ролей.
•	Доступ до особистого кабінету користувача: Сервер має надавати функції перегляду та редагування персональної інформації користувача.
•	Збереження даних: Сервер повинен забезпечити зберігання даних про спортивні зали, користувачів, тренування та умови в залах.
•	Захист даних: Всі важливі атрибути із бази даних мають бути зашифровані та при необхідності дешифровані на серверній частині.
•	CRUD-операції для залів, зон та тренувань: Сервер має надавати можливість перегляду, додавання, редагування та видалення записів з бази даних.
•	Адміністрування: Сервер повинен враховувати наявність ролей, які зможуть мати широкий доступ до даних.

Функції для IoT частини:
•	Збір показників шуму, температури та вологості у залах: Датчики повинні передавати зібрані дані на сервер.

Функції веб-частини:
•	Графічний інтерфейс для реєстрації та авторизації: Користувачам повинна надаватися можливість авторизуватися в системі та створювати облікові записи, використовуючи графічні інтерфейси.
•	Локалізація: Відображення мови та стилів залежно від країни та мови.
•	Налаштування особистого кабінету: Можливість змінювати персональну інформацію.
•	Перегляд інформації про зали, зони та тренування: Можливість додавання нових даних, редагування та видалення.
•	Перегляд поточних показників шуму, температури та вологості: Користувачі зможуть відстежувати поточні умови в реальному часі.

Функції мобільного додатка:
•	Графічний інтерфейс для реєстрації та авторизації: Користувачам повинна надаватися можливість авторизуватися в системі та створювати облікові записи, використовуючи графічні інтерфейси.
•	Локалізація: Відображення мови та стилів залежно від країни та мови.
•	Налаштування особистого кабінету: Можливість змінювати персональну інформацію.
•	Перегляд інформації про зали, зони та тренування: Можливість додавання нових даних, редагування та видалення.

3.2.	Рамки наступних випусків

•	Інтеграція з платіжними системами: Функціонал для обробки платежів буде розроблений у наступних версіях.
•	Розширена локалізація: Первинний випуск буде мати обмежену підтримку локалізацій, з розширенням у майбутньому.
•	Система лояльності: Програмні модулі для управління програмами лояльності та бонусами будуть додані пізніше.
 
4.	Бізнес-контекст
4.3.	Робоче середовище

Система для автоматизації управління спортивними залами та моніторингу умов буде розроблена з використанням сучасних технологій для забезпечення високої доступності, надійності, продуктивності та цілісності даних.
Технологічний стек:
•	Серверна частина: Python, Django
•	База даних: MySQL
•	Веб-частина: HTML5, CSS3, React, JavaScript
•	Мобільна частина: Java
•	IoT застосунок: Мова програмування C
Вимоги до робочого середовища:
1.	Географічна розподіленість користувачів:
o	Система повинна бути доступною для користувачів з різних географічних регіонів, які можуть знаходитися в різних часових поясах.
o	Користувачі повинні мати доступ до системи 24/7, оскільки графік роботи спортивних залів та тренерів може значно варіюватися.
2.	Доступність та надійність:
o	Система має забезпечувати безперебійний доступ, що є критичним для безперервної роботи спортивних залів.
o	Для забезпечення надійності даних необхідне регулярне резервне копіювання та наявність механізмів аварійного відновлення.
3.	Продуктивність:
o	Система повинна забезпечувати швидкий доступ до даних, незалежно від їхнього розташування.
o	Максимальний час відгуку системи не повинен перевищувати 2 секунд для більшості запитів, щоб забезпечити швидке та ефективне обслуговування користувачів.
4.	Масштабованість:
o	Система повинна бути здатна підтримувати зростаючу кількість користувачів та пристроїв.
o	Архітектура системи має бути гнучкою та здатною до масштабування відповідно до зростаючих потреб та обсягів даних.
5.	Безпека та цілісність даних:
o	Доступ до конфіденційної інформації має бути суворо обмежений та контрольований.
o	Всі важливі дані повинні бути зашифровані як при зберіганні, так і під час передачі для запобігання несанкціонованому доступу та зміні.
o	Повинні бути впроваджені засоби шифрування та контролю доступу для забезпечення конфіденційності та цілісності даних.
Ці вимоги та характеристики забезпечать ефективне функціонування системи для управління спортивними залами та моніторингу умов, гарантуючи високу якість обслуговування, безпеку даних та зручність використання для всіх категорій користувачів.
